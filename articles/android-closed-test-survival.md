こんにちは。WebエンジニアのYukiです。

最近、久しぶりに個人開発でAndroidアプリの開発に取り組みました。
フロントエンドに **React Native (Expo)** 、バックエンド（BaaS）として **Supabase** を採用し、さらに **Gemini 2.5 Flash** などの強力なAI APIを組み合わせることで、個人開発のスピードは以前とは比べ物にならないほど劇的に向上しています。

個人開発において最も重要な心得は**「完璧を目指さず、まずは動くものを世に出す」**ことだと思っています。技術の進化のおかげで、今や本気を出せば「1週間でまともなアプリのプロトタイプを作り上げる」ことも可能な素晴らしい時代になりました。

……しかし！
アプリ自体をどれだけスピーディーに作れるようになっても、どうしても個人の力だけではハックできない**「巨大な壁」**が存在していました。

## 絶望の「20人・14日間のクローズドテスト」の壁

久々にAndroidアプリをリリースすべく、ルンルン気分でGoogle Play Consoleを開いたのですが、2023年末から個人開発者向けに導入された**「20人のテスターで14日間のクローズドテスト」**という厳しい制限に直面しています……（現在は一部緩和され12人に変更されているという話もありますが、それでも個人には厳しい壁です！）。

私の周りにAndroidユーザーがそんなにいるはずもなく、リリースを目前にして完全に途方に暮れています😭

そこで、この記事を読んでくださっている皆様に**「相互テスト（互助）」**のお願いです！

私のアプリをテスト（14日間のインストール）していただける方がいらっしゃいましたら、**私も全力であなたのアプリをテストさせていただきます！**
（もちろん、現在ご自身のアプリをテスト中でない方からの、純粋な人助け・ご協力も大歓迎です🙇‍♂️）

## 📱 今回テストをお願いしたいアプリ「Respo（レスポ）」

今回私がリリースを目指しているのは、こんなアプリです。

* **アプリ名**: Respo（レスポ）
* **概要**: 「管理しない」健康パートナー。毎日のカロリー計算や厳しい指導で「ウェルネス燃え尽き」を起こした人の自己肯定感を高める、優しいAIチャットボット＆ライフログアプリです。

### ちょっとだけ技術的なお話（テックスタック）

開発には **React Native (Expo SDK 54)** と **Supabase** を採用しています。
特にこだわったのは、**Gemini 2.5 Flash API を使った「写真を撮るだけの食事記録」と「優しいAIチャット」**です。AIには決して厳しい指導をさせず、ユーザーの疲労に共感し、「〜してみませんか？」と優しく提案・伴走してくれるようにプロンプトチューニングを行っています。
また、Androidの **Health Connect API** と連携し、歩数や睡眠データを自動で取り込むことで、ユーザーの入力の手間を極力省く設計にしています。
