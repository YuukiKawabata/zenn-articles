2026年1月、デザインプラットフォーム大手のFigmaは、Anthropicの「Claude」と連携し、FigJam内でダイアグラムを自動生成する新機能を発表しました。

ただの「AI機能の追加」だと思っていませんか？

実際に調査を進めていく中で、これは単なる機能追加ではなく、エンタープライズにおける**「思考の可視化」のあり方を根本から変えるパラダイムシフト**であると確信しました。
これまで独立していた「対話型AI」と「ビジュアルコラボレーション」が、**Model Context Protocol (MCP)** という技術によってシームレスに統合されたのです。

この記事では、このインテグレーションの技術的背景、PDE（Product, Design, Engineering）チームへの影響、そして競合他社（Miro等）との比較まで、包括的な分析をお届けします。

## 1. エンタープライズAIのパラダイムシフト

これまで、生成AIを業務で使うと言えば、ChatGPTやClaudeのチャット画面でテキストを打ち込むのが主流でした。しかし、今回のFigmaとAnthropicの提携は、**「AIがユーザーの作業するライブスクリーン上で直接アクションを実行する」**という、エージェント型ワークフローへの大きな一歩です。

Claudeは、テキストだけでなく、画像やPDFといった資料を読み込み、それをFigJam上の構造化された図（フローチャートやマインドマップなど）として出力します。

## 2. PDEワークフローの変革

プロダクトマネージャー（PM）、デザイナー、エンジニアのそれぞれの視点で、どのようなメリットがあるのか見ていきましょう。

### プロダクトマネジメント：要件定義の可視化

PMにとって、PRD（要求仕様書）をステークホルダーに分かりやすく伝えるのは骨の折れる作業です。
この新機能を使えば、**長文のPRD（PDFなどの資料）をClaudeに読み込ませるだけで、ガントチャートやタイムラインの叩き台をFigJam上に一瞬で生成**できます。

テキストの束が視覚的なマップに変換されることで、スケジュールの無理な点や依存関係の矛盾に早期に気づくことができます。

### デザイン：体験設計の加速

デザイナーにとって、ユーザージャーニーマップや情報アーキテクチャ（IA）をゼロから作るのは、大きな認知負荷がかかります（いわゆるコールドスタート問題）。
「ユーザーがアカウントを作成し、商品を検索して購入に至るまでのフローを作成して」とClaudeに頼むだけで、基本的なフローチャートが生成されます。デザイナーは、**図形の配置という単調な作業から解放され、体験の洗練（Refine）に時間を使える**ようになります。

### エンジニアリング：設計と実装の乖離を防ぐ

エンジニアにとっては、アーキテクチャの可視化に役立ちます。シーケンス図やステートマシン図を生成することで、実装前にシステム構造をチームで共有できます。
さらに、**Figma MCPサーバー**を介してIDE（開発環境）と連携すれば、FigJamの図を基に、**そのプロジェクトのデザインシステムに準拠したコード**をClaudeに生成させることも可能です。これが「設計と実装の乖離」を防ぐ強力な武器になります。

## 3. 機能詳細とマルチモーダル処理

### 入力と出力の多様性

*   **入力**: テキストプロンプトはもちろん、手書きメモの写真、スクショ、PDFなど。
*   **出力**: フローチャート、デシジョンツリー、ガントチャート、シーケンス図、ステート図など。

Claudeはコンテキストを理解し、最適な図解形式を提案してくれます。

### チャットUI vs Claude Code (CLI)

*   **ブラウザ版Claude**: チャット内で図のプレビュー（Artifact）を作成し、微調整してからFigJamにエクスポートできます。
*   **Claude Code (CLI)**: エンジニア向け。CLIから直接FigJamのURLを生成し、ブラウザで開いて編集するストレートな動線です。

## 4. 技術の要：Model Context Protocol (MCP)

この連携の裏側にあるのが、Anthropicが提唱する **Model Context Protocol (MCP)** です。MCPは、AIが外部のデータやツールと安全に接続するための標準規格です。

Figmaは「Figma MCPサーバー」を提供しており、これによってAIがFigma/FigJamの内部データ構造（ノード、プロパティなど）を「理解」できるようになります。

*   **リモートMCPサーバー**: 設定が簡単。ブラウザ利用者向け。
*   **デスクトップMCPサーバー**: ローカルアプリ経由。**「選択している要素（Selection）」に基づいた指示**が出せるのが強力。

## 5. ガバナンスとセキュリティ

企業導入で気になるのがセキュリティですが、AIからのアクセス権限は、Figmaユーザー個人の権限と同期しています。自分が閲覧権限を持たないファイルには、AI経由でもアクセスできません。

また、プランに応じてAPIコール数（ツール利用回数）の制限があります。
*   **Enterprise**: 1日600回
*   **Organization/Professional**: 1日200回
*   **Starter**: 月6回（お試し枠）

実業務でバリバリ使うなら、有料プランが必須という設計です。

## 6. 競合（Miro）との比較

オンラインホワイトボードの王者 **Miro** との対比も興味深いです。

*   **Miro**: 「Miro Assist」による付箋の構造化や要約に強み。多機能でテンプレートも豊富。
*   **FigJam**: これまでは「楽しさ」「シンプルさ」重視でしたが、Claudeの頭脳を手に入れたことで、**「複雑な論理構造化」も守備範囲**になりました。

Figmaエコシステム内で完結できる利便性は、開発チームにとって強力なアドバンテージです。

## 7. まとめ

Figma × Claudeの連携は、**「人間が考え、AIが描く」**（あるいはその逆）という新しい共同作業の形を提示しています。

*   **思考のオフロード**: 図解作成の手間をゼロに。
*   **FigJamの進化**: シンプルなUIのまま、高度なロジック処理が可能に。
*   **エージェント型AIの実用化**: チャット画面の外で仕事をするAIの先駆け。

私たちの仕事は、「図を書くこと」から「図を見て議論し、意思決定すること」へとシフトしていくでしょう。ぜひ一度、手元の資料をClaudeに投げて、FigJamで可視化してみてください。
